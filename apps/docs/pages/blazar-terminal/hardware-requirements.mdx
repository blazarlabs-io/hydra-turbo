# Hardware Requirements

The Blazar Terminal is designed to run on Raspberry Pi Zero 2W hardware, providing a cost-effective and reliable platform for payment processing.

## Primary Hardware

### Raspberry Pi Zero 2W

The **Raspberry Pi Zero 2W** serves as the main computing platform for the Blazar Terminal:

| Specification | Value |
|---------------|-------|
| **Processor** | ARM Cortex-A53 quad-core 64-bit @ 1.0GHz |
| **Memory** | 512MB LPDDR4 SDRAM |
| **Storage** | microSD card slot (≥8GB Class 10 recommended) |
| **Connectivity** | Wi-Fi 802.11 b/g/n (2.4GHz) + Bluetooth 4.2 |
| **GPIO** | 40-pin header (standard Pi layout) |
| **Power** | 5V/2.5A via microUSB or GPIO pins |
| **Dimensions** | 65 × 30 × 5mm (2.56 × 1.18 × 0.20 inches) |
| **Weight** | 11g (0.39 oz) |

### Storage Requirements

**Minimum Storage**: 8GB microSD card (Class 10 or better)

**Recommended Storage**: 16GB or larger for:
- Operating system
- Application files
- Log files
- System updates
- Backup data

**Storage Classes**:
- **Class 10**: Minimum requirement for reliable operation
- **UHS-I**: Recommended for better performance
- **A1/A2**: Optimal for application performance

## Hardware Setup

### 1. SD Card Preparation

**Flash Raspberry Pi OS Lite (64-bit recommended)**

```bash
# Download Raspberry Pi Imager
# Flash OS Lite (64-bit) to microSD card
# Enable SSH and Wi-Fi during initial setup
```

**Essential Configuration**:
- Enable SSH access
- Configure Wi-Fi credentials
- Set static IP address (optional but recommended)
- Update system packages

### 2. Bluetooth Configuration

**Enable Bluetooth Service**

```bash
sudo systemctl enable bluetooth
sudo systemctl start bluetooth
sudo usermod -a -G bluetooth $USER
```

**Bluetooth Configuration File** (`/etc/bluetooth/main.conf`):

```ini
[General]
ControllerMode = dual
DiscoverableTimeout = 0
PairableTimeout = 0
AutoEnable = true
```

**Set Bluetooth Capabilities**:

```bash
# Allow BLE advertising without authentication
sudo setcap cap_net_raw+eip $(which node)
```

### 3. Network Configuration

**Wi-Fi Setup**:
- Ensure stable 2.4GHz Wi-Fi connection
- Configure static IP for reliable merchant app connections
- Test network connectivity to Blazar API endpoints

**Port Configuration**:
- Ensure port **3000** is available for WebSocket server
- Configure firewall rules if needed
- Test port accessibility from merchant applications

### 4. Power Management

**Power Requirements**:
- **Input**: 5V/2.5A via microUSB or GPIO pins
- **Consumption**: ~2.5W active, ~1.5W idle
- **UPS Support**: Recommended for critical deployments

**Power Supply Options**:
- **USB Power Adapter**: 5V/2.5A wall adapter
- **Power Bank**: Portable battery pack for mobile use
- **UPS System**: Uninterruptible power supply for critical deployments

## Hardware Verification

### System Check Commands

```bash
# Check system information
uname -a
cat /proc/cpuinfo
free -h
df -h

# Verify Bluetooth
sudo hcitool dev
sudo hcitool lescan

# Test network connectivity
ping -c 4 8.8.8.8
curl -I https://api.blazarpay.com

# Check available ports
netstat -tulpn | grep :3000
```

### Performance Testing

**CPU Performance**:
```bash
# Stress test CPU
stress --cpu 4 --timeout 60s
```

**Memory Usage**:
```bash
# Monitor memory usage
htop
free -h
```

**Storage Performance**:
```bash
# Test storage speed
dd if=/dev/zero of=testfile bs=1M count=100
dd if=testfile of=/dev/null bs=1M
rm testfile
```

## Troubleshooting

### Common Hardware Issues

**Power Problems**:
```bash
# Check power supply
vcgencmd get_throttled
vcgencmd measure_temp
```

**Bluetooth Issues**:
```bash
# Reset Bluetooth
sudo systemctl restart bluetooth
sudo hciconfig hci0 reset
```

**Network Connectivity**:
```bash
# Check network configuration
ip addr show
ip route show
systemctl status networking
```

**Storage Issues**:
```bash
# Check disk health
sudo fsck /dev/mmcblk0p2
df -h
```
